2. 第一步：把小数问题变成大整数问题

令内部精度 D=1000256，定义固定缩放因子：

S=10D=100500128纯文本：S=10^D=100^500128。它不是实际生成的一串 1 后跟零，而是数组中约定的小数点位置。

任何非负实数 x 都用整数 X 表示：

X=⌊xS⌋,x≈XS纯文本：把 x 乘以 10^D 后取整数。数组始终保存 X，程序心里记着它代表 X/S。

于是加减就是整数加减；两个定点数相乘时，要把多出来的一份缩放除掉：

Z=⌊XYS⌋≈(xy)S纯文本：X≈xS、Y≈yS，所以 XY≈xyS²；除以 S 后重新得到代表 xy 的同尺度整数。

先用 8 位小数做一个完整缩小版

假设只保留 8 位，S=10⁸=100⁴。那么：

floor(π × 10⁸) = 314159265

把它按 100 进制连续取余，得到小端数组 [65, 92, 15, 14, 3]。反过来从整数位向小数位读，就是：

3 . 14 15 92 65　→　3.14159265

正式数组有 500128 个小数 limb，每个保存两位：前 500000 个形成一百万个目标位，后 128 个形成 256 个保护位。另有两个整数 limb，用来容纳迭代中的 2、3 等常量。

    def __init__(self, digits, guard_digits, device="cuda:0"):
        if digits <= 0 or guard_digits < 0:
            raise ValueError("digits must be positive and guard_digits non-negative")
        self.digits = int(digits)
        self.guard_digits = int(guard_digits)
        # 每个 limb 保存两位；向上取整得到所需小数 limb 数
        self.fractional_limbs = (digits + guard_digits + 1) // 2
        self.integer_limbs = INTEGER_LIMBS
        # 完整数组 = 小数 limb + 两个整数 limb
        self.limb_count = self.fractional_limbs + self.integer_limbs
        # 后续所有多精度数组都绑定到同一计算设备
        self.device = torch.device(device)
        if self.device.type != "cuda":
            raise ValueError("the numerical core requires a CUDA device")
        self._indices = {}
        # 在设备上累计所有 FFT 乘法的最大舍入误差
        self.max_fft_residual = torch.zeros((), dtype=torch.float64, device=self.device)
        self.operations = Counter()
        self.max_fft_length = 0
  1. (digits + guard_digits + 1) // 2 正是“两个十进制位一个 limb”。正式参数代入后得到 500128。
  2. limb_count 再加两个整数 limb；它们和小数 limb 组成一条定长数组，小数点只是约定在第 fractional_limbs 格之后。
  3. device.type != "cuda" 的检查不参与数学，但保证后续这些数组运算不会悄悄退回普通 Python 数字。

    def literal(self, text):
        """Encode a short non-negative decimal literal (initialization only)."""
        text = str(text).strip()
        if text.startswith("-"):
            raise ValueError("this implementation stores unsigned values")
        if "." in text:
            integer, fraction = text.split(".", 1)
        else:
            integer, fraction = text, ""
        if not integer:
            integer = "0"
        if not integer.isdigit() or (fraction and not fraction.isdigit()):
            raise ValueError("invalid decimal literal: %r" % text)
        if len(fraction) > 2 * self.fractional_limbs:
            discarded = fraction[2 * self.fractional_limbs :]
            if any(ch != "0" for ch in discarded):
                raise ValueError("literal does not fit fixed-point precision")
            fraction = fraction[: 2 * self.fractional_limbs]

        host_limbs = [0] * self.limb_count
        # 在右侧补零，相当于把 0.25 写成 0.250000…
        fraction = fraction.ljust(2 * self.fractional_limbs, "0")
        # 从小数点向右，每两个十进制字符组成一个 base-100 limb
        for pair_index in range(self.fractional_limbs):
            pair = fraction[2 * pair_index : 2 * pair_index + 2]
            # 靠近小数点的高位放到更高下标，形成小端布局
            host_limbs[self.fractional_limbs - 1 - pair_index] = int(pair)
        integer_value = int(integer)
        # 整数部分同样按 base 100 连续取余
        for offset in range(self.integer_limbs):
            host_limbs[self.fractional_limbs + offset] = integer_value % BASE
            integer_value //= BASE
        if integer_value:
            raise OverflowError("integer literal exceeds the two integer limbs")
        # 编码完毕后一次性转成 CUDA int64 limb 数组
        return torch.tensor(host_limbs, dtype=torch.int64, device=self.device)
  1. fraction.ljust(..., "0") 在字面量右侧补零，相当于把 0.25 写成 0.250000…，并没有改变数值。
  2. 循环每次读取两个字符；fractional_limbs - 1 - pair_index 把最靠近小数点的 25 放到高小数 limb，把更低位放到更低下标，因此形成小端布局。
  3. 整数部分从 self.fractional_limbs 开始按 100 连续取余；这就是前面示例里把 314159265 拆成 [65,92,15,14,3] 的同一动作。
  4. 最后一行只把已经编码好的 limb 放到计算设备；之后 Newton 和 Gauss 看到的始终是这种定点数组。

3. 第二步：一百万位整数怎样相乘

Newton 和 Gauss 迭代几乎都可以归结为乘法。把 base-100 数组看成多项式：

A(x)=a0+a1x+a2x2+⋯,x=100纯文本：数组元素是多项式系数；把 x 换成 100，恰好恢复原来的大整数。

乘积第 k 个原始系数是所有满足 i+j=k 的 a[i]×b[j] 之和，这叫卷积。例如：

0.1234  → limbs [34, 12]
0.5678  → limbs [78, 56]

原始卷积：
c0 = 34×78            = 2652
c1 = 34×56 + 12×78    = 2840
c2 = 12×56            =  672

按 base 100 进位：[2652,2840,672] → [52,66,0,7]
完整乘积：0.07006652

若直接做所有配对，需要约 N² 次乘法。FFT 把卷积变成“变换—逐点相乘—逆变换”，工作量降到约 N log N，因此百万位才实际可算。

对应源码：FFT 卷积与定点缩放

pi_gpu.py
    def convolve_limbs(self, left, right):
        """Exact limb convolution via float64 FFT plus checked integer rounding."""
        self._check_pair(left, right, require_context_length=False)
        # 长度 n 和 m 的多项式卷积共有 n+m-1 个原始系数
        conv_length = left.numel() + right.numel() - 1
        # One additional zero coefficient is required for the final normalized
        # carry (e.g. 99 * 99 is a two-limb result although its raw convolution
        # has one coefficient).
        # 再留一格接收最高位的 base-100 进位
        product_length = left.numel() + right.numel()
        # 补零到 2 的幂，避免 FFT 的循环卷积绕回低位
        fft_length = _next_power_of_two(product_length)
        # 补零到 2 的幂，避免 FFT 的循环卷积绕回低位
        self.max_fft_length = max(self.max_fft_length, fft_length)
        # 把第一个 limb 序列变换到频域
        left_fft = torch.fft.rfft(left.to(torch.float64), n=fft_length)
        if left.data_ptr() == right.data_ptr() and left.numel() == right.numel():
            right_fft = left_fft
        else:
            # 第二个序列也做同样变换
            right_fft = torch.fft.rfft(right.to(torch.float64), n=fft_length)
        # 频域逐点相乘后逆变换，得到近似卷积系数
        raw = torch.fft.irfft(left_fft * right_fft, n=fft_length)
        # 理论系数是整数，因此舍入到最近整数
        rounded_float = torch.round(raw)
        # 检查逆 FFT 结果离整数有多远
        residual = torch.max(torch.abs(raw - rounded_float))
        # 检查逆 FFT 结果离整数有多远
        self.max_fft_residual = torch.maximum(self.max_fft_residual, residual)
        _async_assert(residual.le(FFT_RESIDUAL_LIMIT), "FFT rounding residual exceeded 0.1")
        # 从 float64 转回真正的 int64 系数
        rounded = rounded_float.to(torch.int64)
        if conv_length < fft_length:
            rounded[conv_length:] = 0
        _async_assert(torch.all(rounded >= 0), "FFT rounded to a negative coefficient")
        # 计算最坏卷积系数上界，供进位归一化使用
        coefficient_bound = (BASE - 1) ** 2 * min(left.numel(), right.numel())
        # 完成 base-100 进位，使每格重新落到 0…99
        result = self.normalize_nonnegative(rounded, coefficient_bound)
        self.operations["fft_multiply"] += 1
        self.operations["rfft"] += 1 if right_fft is left_fft else 2
        self.operations["irfft"] += 1
        return result[:product_length]
  1. conv_length = len(A)+len(B)-1 是普通多项式乘积的系数数目；额外的 product_length 格用来接收最高位进位。
  2. rfft 把两个 limb 序列变到频域，left_fft * right_fft 是逐点相乘，irfft 把它还原成卷积系数。这里并没有把 π 变成频率；FFT 只是快速完成大整数乘法。
  3. torch.round(raw) 把浮点 FFT 的近似结果还原成整数系数。紧接着计算的 residual 是“离最近整数还有多远”；超过 0.1 便立即判错。
  4. normalize_nonnegative 完成 base-100 进位，使每格重新落在 0…99。传入的上界说明最坏系数也在可安全舍入范围内。
  5. 左右参数指向同一张量时就是平方；代码复用一次正向 FFT，但数学结果仍是同一个卷积。
pi_gpu.py
    def mul(self, left, right):
        self._check_pair(left, right)
        # 先得到完整整数积 XY，它代表 xyS²
        normalized = self.convolve_limbs(left, right)
        # 跳过最低 F 个 limb，等价于除以 S=100^F
        start = self.fractional_limbs
        # 只保留固定精度所需的同样长度
        stop = start + self.limb_count
        result = torch.zeros(self.limb_count, dtype=torch.int64, device=self.device)
        # 这是 floor(XY/S) 对应的 limb 区间
        available = normalized[start : min(stop, normalized.numel())]
        # 复制到新的定长定点数组，供下一轮继续使用
        result[: available.numel()] = available
        if normalized.numel() > stop:
            _async_assert(torch.all(normalized[stop:] == 0), "fixed-point multiplication overflow")
        return result
  1. convolve_limbs 得到的是代表 XY≈xyS² 的完整整数积。
  2. start = self.fractional_limbs 后从第 F 个 limb 开始截取，正好丢掉低端的 100^F=S 因子，也就是执行 floor(XY/S)。
  3. 截取后的数组长度重新回到 limb_count，所以它可直接送入下一轮乘法；这就是所有迭代始终保持同一百万位尺度的原因。

4. 第三步：不用长除法，怎样求倒数和平方根

倒数：误差真的会平方

想求 1/x，先有一个很粗的猜测 y，再反复执行：

y′=y(2−xy)纯文本：新猜测 y'=y(2−xy)。右边只有乘法和减法。

令当前相对误差 e=1−xy。代入上式可直接算出：

1−xy′=(1−xy)2=e2纯文本：更新后的误差正好是原误差的平方。0.01 变成 0.0001，正确位数约翻倍。

平方根：先求倒平方根

Gauss 迭代需要 √(ab)。程序先令 y≈1/√x，使用：

y′=y(3−xy2)2,x≈xy纯文本：迭代得到 y≈1/√x，最后乘一次 x 得到 √x。

以 x=2、y₀=1 为例，序列是 1 → 0.5 → 0.625 → 0.693359375 → 0.706708468… → 0.707106444…，迅速逼近 1/√2=0.707106781…。正式程序每次执行 24 轮；每一轮的乘法都是上一节的百万位定点乘法。

对应源码：reciprocal / inverse_sqrt / sqrt

pi_gpu.py
    def reciprocal(self, value, iterations=24):
        """Newton reciprocal y <- y(2-xy), for the final denominator near one."""
        self._check(value)
        y = self.literal("1")
        # 常量 2 与 x、y 使用完全相同的定点尺度
        two = self.literal("2")
        for _ in range(iterations):
            # 这一行逐字实现 y ← y(2-xy)
            y = self.mul(y, self.sub(two, self.mul(value, y)))
            self.operations["reciprocal_iteration"] += 1
        return y
    def inverse_sqrt(self, value, iterations=24):
        """Newton inverse square root y <- y(3-xy^2)/2."""
        self._check(value)
        y = self.literal("1")
        # 倒平方根公式需要同尺度常量 3
        three = self.literal("3")
        for _ in range(iterations):
            # 先计算 y²
            y_squared = self.mul(y, y)
            # 再形成修正量 3-xy²
            correction = self.sub(three, self.mul(value, y_squared))
            # 计算 y(3-xy²)/2，误差在此近似平方
            y = self.div2(self.mul(y, correction))
            self.operations["inverse_sqrt_iteration"] += 1
        return y
    def sqrt(self, value, iterations=24):
        # sqrt(x)=x×(1/sqrt(x))
        return self.mul(value, self.inverse_sqrt(value, iterations))
  1. 倒数函数从定点常量 y=1 开始。内层 self.mul(value, y) 是 xy，self.sub(two, ...) 是 2−xy，外层乘法于是完整对应 y(2−xy)。
  2. 倒平方根先算 y_squared=y²，再算 correction=3−xy²，最后执行 y×correction/2；变量名就是公式中的每一项。
  3. div2 不是降低精度的普通浮点除法，而是在 base-100 定点数组上精确做整数除 2。
  4. sqrt 只做最后一步 x×(1/√x)。因此代码从未实现复杂的百万位长除法或逐位开方，全部复用了已经解释过的乘法。
  5. iterations=24 是固定上限；由于误差反复平方，后面的轮次一次修正成倍增加的正确位。

5. 核心：Gauss–Legendre 为什么会算出 π

四个变量从哪里开始

a₀ = 1
b₀ = 1/√2
t₀ = 1/4
p₀ = 1

a 与 b 分别做算术平均和几何平均。对正数而言，几何平均不大于算术平均，所以二者会从上下两侧快速靠近同一个极限，这个极限叫 AGM（算术—几何平均）。

an+1=an+bn2,bn+1=anbn纯文本：下一轮 a 是算术平均，b 是几何平均。

t 累计每轮两个平均数之间的“差距能量”，p 每轮加倍：

tn+1=tn−pn(an−an+1)2,pn+1=2pn纯文本：t 减去加权平方差，p 变成两倍。

每一轮都可以形成一个 π 的近似：

πn=(an+bn)24tn纯文本：πₙ=(aₙ+bₙ)²/(4tₙ)。程序最后实际计算的就是这一式。

把第一轮逐项算出来

先暂时只看十几位小数，正式程序只是把每一行换成拥有 1000256 位的定点整数运算：

b₀ = 1/√2
   = 0.7071067811865475…

a₁ = (a₀+b₀)/2
   = (1+0.7071067811865475…)/2
   = 0.8535533905932737…

b₁ = √(a₀b₀)
   = √(1×0.7071067811865475…)
   = 0.8408964152537145…

t₁ = t₀-p₀(a₀-a₁)²
   = 0.25-1×(1-0.8535533905932737…)²
   = 0.2285533905932737…

p₁ = 2

π₁ = (a₁+b₁)²/(4t₁)
   = 3.140579250522168…

第一轮还不够准，但它展示了数字的真实来源：没有查表，只有平均、乘法、平方根、平方和一次除法。下一轮把这里得到的 a₁,b₁,t₁,p₁ 原样代回同一组公式。

为什么这不是“碰巧接近 π”

背后的桥梁是完全椭圆积分：

K(k)=∫0π2dθ1−k2sinθ2=π2AGM(1,1−k2)纯文本：Gauss 证明，椭圆积分 K 可以用算术—几何平均 AGM 表示，而式中明确含有 π。

选择 k=1/√2 时，它的互补参数仍是 1/√2。Legendre 恒等式把这个对称情形中的两个椭圆积分合并；上面由平方差累计出来的 t 正好记录其中第二类积分的修正项。最终得到严格恒等式 π=M²/t∞，其中 M=AGM(1,1/√2)。当 a、b 都趋于 M 时，(a+b)²/(4t) 就趋于 π。也就是说，初始化中的 1/√2、每轮平方差和最后的除法是一套完整数学恒等式，不是经验拟合。

pi_gpu.py
def compute_pi_digits(digits, guard_digits, device="cuda:0", gauss_iterations=22,
                      newton_iterations=24):
    """Return (CUDA uint8 decimal digits, arithmetic context)."""
    # 建立统一的 D 位定点运算环境
    context = FixedPointCUDA(digits, guard_digits, device=device)
    # 精确写入 a0=1，并由它得到 1/2
    one = context.literal("1")
    half = context.div2(one)
    a = one
    # b0=sqrt(1/2)=1/sqrt(2)
    b = context.sqrt(half, newton_iterations)
    # 精确写入 t0=1/4
    t = context.literal("0.25")
    p = 1

    # 下面四个更新固定重复 22 轮
    for _ in range(gauss_iterations):
        # 算术平均：a'=(a+b)/2
        a_next = context.div2(context.add(a, b))
        # 几何平均：b'=sqrt(ab)
        b_next = context.sqrt(context.mul(a, b), newton_iterations)
        # 差值 a-a' 决定本轮 t 的修正量
        difference = context.sub(a, a_next)
        # 计算 p(a-a')²
        correction = context.mul_small(context.mul(difference, difference), p)
        # t'=t-p(a-a')²
        t = context.sub(t, correction)
        a, b = a_next, b_next
        # 下一轮平方差的权重加倍
        p *= 2
        context.operations["gauss_legendre_iteration"] += 1

    # 最终公式的分子：(a+b)²
    numerator = context.mul(context.add(a, b), context.add(a, b))
    # 最终公式的分母：4t
    denominator = context.mul_small(t, 4)
    # 除以 4t 改写成乘以 Newton 求出的倒数
    pi_value = context.mul(numerator, context.reciprocal(denominator, newton_iterations))
    # 把 π 的定点数组展开成目标十进制数字
    return context.decimal_digits(pi_value, digits), context
  1. one、half、a、t 和 p 对应初始化表。b=context.sqrt(half) 正是 b₀=√(1/2)=1/√2。
  2. a_next 和 b_next 是算术平均、几何平均两行公式；这里的 context.mul(a,b) 先得到 ab，再交给刚才解释的 Newton 平方根。
  3. difference=a−a_next，接下来平方并乘 Python 小整数 p，得到 p(a−a')²；t=context.sub(t,correction) 完成 t 的更新。
  4. p *= 2 只是更新下一轮的权重。p 本身很小，所以乘它不需要再做一次 FFT。
  5. 循环结束后，numerator=(a+b)²，denominator=4t；先用 Newton 得到分母倒数，再与分子相乘，严格对应 (a+b)²/(4t)。
  6. 函数最后没有生成字符串，而是调用 decimal_digits 返回仍在计算设备上的数字张量。下一节解释它怎样成为恰好一百万个字符。

6. 把所有部分接起来：从 1 到一百万位

  1. 确定小数点位置。设 D=1000256，所有数都存成约等于真实值乘 S=10^D 的 base-100 整数数组。数组有 500128 个小数 limb 和 2 个整数 limb。
  2. 写入精确常量。a=1、t=1/4、p=1 都能在十进制定点中精确表示；1/2 也精确。
  3. 算出 b₀。对 x=1/2 做 24 轮倒平方根迭代，再乘 x，得到 √(1/2)=1/√2 的 1000256 位定点近似。
  4. 执行 22 轮。每轮先算 (a+b)/2，再用 Newton 算 √(ab)，然后用 t←t−p(a−a')² 修正 t，最后令 p←2p。每个乘号都走 FFT 定点乘法。
  5. 形成分子和分母。分子 N=(a+b)²；分母 Q=4t。此时 N/Q 已在 1000256 位范围内逼近 π。
  6. 把除法改成乘法。对 Q 做 24 轮 y←y(2−Qy) 得到 y≈1/Q，再算 P=N×y。P 就是 π 的最终定点数组。
  7. 把 limb 变成字符。从整数小数点旁边开始反向读取 500000 个 limb；每个值 v 输出 v//10 和 v%10 两个数字。这样正好得到 1,000,000 个数字。
  8. 保护位不输出。最深处 128 个 limb 只承受前面多次截断造成的误差，最终直接丢弃。结果于是是目标前缀，而不是被边界舍入污染的尾部。
# 数学伪代码；每个变量实际上都是长度相同的定点 limb 数组
a = 1
b = sqrt(1/2)                 # Newton 倒平方根
t = 1/4
p = 1

repeat 22 times:
    next_a = (a + b) / 2
    next_b = sqrt(a * b)      # Newton + FFT 定点乘法
    t = t - p * (a - next_a)^2
    a, b = next_a, next_b
    p = 2 * p

pi_fixed = (a + b)^2 * reciprocal(4 * t)
digits = 从 pi_fixed 依次拆出前 1,000,000 个小数数字

pi_gpu.py
    def decimal_digits(self, value, count):
        """Return fractional decimal digits as a CUDA uint8 tensor."""
        self._check(value)
        # 每个 limb 提供两位，所以一百万位需要 500,000 个 limb
        pairs = (int(count) + 1) // 2
        if pairs > self.fractional_limbs:
            raise ValueError("requested digits exceed fixed-point precision")
        # 取小数点旁的目标 limb；更深的保护位不在切片中
        # 存储是小端，显示前先反转成从高位到低位
        pair_values = torch.flip(
            value[self.fractional_limbs - pairs : self.fractional_limbs], dims=(0,)
        )
        # 为逐个十进制数字分配 uint8 输出张量
        digits = torch.empty(2 * pairs, dtype=torch.uint8, device=self.device)
        # 每个 limb 的十位写到第 1、3、5…个位置
        digits[0::2] = torch.div(pair_values, 10, rounding_mode="floor").to(torch.uint8)
        # 每个 limb 的个位写到第 2、4、6…个位置
        digits[1::2] = torch.remainder(pair_values, 10).to(torch.uint8)
        # 若请求奇数位，在这里裁掉最后多出的一位
        return digits[:count]
  1. pairs=(count+1)//2 把目标数字数换成 limb 数；一百万位正好是 500,000 对。
  2. 数组是小端的，所以切片先取小数点下方所需的 500,000 格，再用 torch.flip 反转成阅读顺序。更深的 128 个保护 limb 没进入这个切片。
  3. digits[0::2]=pair_values//10 取每个 limb 的十位，填入第 1、3、5…个位置；digits[1::2]=pair_values%10 取个位，填入第 2、4、6…个位置。
  4. 结果类型是 uint8，每个元素仍是数值 0…9。通过逐位比较后，CPU 才给每项加字符 '0' 并写成文本。
  5. 因此第 1,000,000 位对应最后一个 pair_values 的 %10 分支，这正是第 500,000 个目标 limb 的个位。
现在可以精确回答“第 1,000,000 位怎么来的”：它是最终 pi_fixed 数组中第 500,000 个目标小数 limb 的个位。该 limb 是 22 轮 Gauss 更新中所有定点加减、FFT 乘法和 Newton 平方根共同算出的。本次该数字是 1。

7. 怎样确认这一百万位不是“看起来像对”

工程验收只保留与数值可信度直接相关的三件事：

  1. 256 个保护位：所有运算内部多算这些位，输出时丢弃。
  2. FFT 可舍入性：最大观测残差是 1.07288360596e-06，小于验收阈值 0.1，更远小于舍错整数所需的 0.5。
  3. 两个独立参考：两份参考先互比，再分别与计算结果在计算设备上逐位比较；mismatch 分别为 [0, 0]。
验收结论：计算结果 mismatch=0，参考之间 mismatch=0，状态为 passed。
