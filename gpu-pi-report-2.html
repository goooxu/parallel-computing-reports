圆周未央，万核同数

在 GPU 上使用 PyTorch 计算 π 的小数点后 100 万位

一句话总结。 只用 PyTorch，在一块 GPU 上从零实现「大整数运算」， 再用收敛极快的 Chudnovsky 公式把 π 算到小数点后 100 万位，与公开权威数据逐位比对——全部一致。 本文按「原理 → 对应代码 → 详解」的顺序展开。

目录

1. 挑战：为什么不能直接算 2. 并行的唯一心法：从循环到整片数组 3. 用 limb 数组表示大整数（代码） 4. 加法与进位（代码） 5. Chudnovsky 公式：每项顶 14 位（代码） 6. 二分裂：把求和变成少数几次乘法（代码） 7. FFT 大整数乘法（代码） 8. 牛顿迭代：除法与开方（代码） 9. 组装出 π（代码） 10. 与公开 ground truth 比对（代码 + 结果）

1. 挑战：为什么不能直接算

浮点数装不下。 C++ 的 double、Python 的 float 都是 64 位浮点数，只有约 15～16 位有效数字。 要 100 万位，差了 5 个数量级——这不是精度不够，而是类型里根本没那么多位置。

PyTorch 里没有「大整数」。 Python 内置 int 虽是任意精度，但它在 CPU 上单线程运行； 而 GPU 张量只有 int64、float64 这类定长类型。所以我们要自己用一大片小数字（数组）拼出一个大数， 把大数的加减乘除翻译成对整片数组的运算，交给 GPU 上成千上万个核心并行完成。

2. 并行的唯一心法：从「循环」到「整片数组」

CPU＝几位博士生
核心少但强，擅长「一步依赖上一步」的复杂逻辑；跑 for 循环是一次一格。
GPU＝上万名小学生
GPU 上数万个 CUDA 核心，每个只会简单算术，但能同时开工。

要用上 GPU，就得把「逐元素的循环」改写成「对整片数组的一次运算」：

左：for i: a[i]=b[i]+c[i]，n 步顺序。右：a = b + c，一次并行完成。
全文最重要的一句话：后面每一个大数运算，我们都会把它写成「对整片 limb 数组的运算」，这样它才真正跑在 GPU 上。

3. 用 limb 数组表示大整数

原理。 把大整数拆成许多 0～999 的小数字，每个叫一个 limb，整体是一个 int64 数组（低位在前）。 选基 1000（＝10³）是因为最终要输出十进制：每个 limb 恰好 3 位数字，输出零成本。100 万位 π ≈ 33.4 万个 limb。

大整数 = limb 数组（小端，下标 0 是最低位）。

代码。 构造与输出——注意输出时只是把每个 limb 补足 3 位再拼接，天然是十进制：

def from_int(n):
    """Python 整数 -> limb 张量（要求 n >= 0，通常 n 较小，如常数/种子）。"""
    n = int(n)
    assert n >= 0
    if n == 0:
        return zero()
    limbs = []
    while n > 0:
        limbs.append(n % B)
        n //= B
    return torch.tensor(limbs, dtype=DT, device=DEV)
def to_decimal_str(x):
    """limb 张量 -> 十进制字符串（无前导零）。每个 limb 补足 3 位再拼接。"""
    limbs = x.tolist()
    s = "".join("%03d" % int(d) for d in reversed(limbs))
    s = s.lstrip("0")
    return s if s else "0"

详解。 from_int 反复取 n % 1000 当作最低 limb、再 n //= 1000，得到小端数组； 它只用于常数和种子这类小数字。to_decimal_str 把数组从高位到低位、每个 limb 用 %03d 补零拼接， 再去掉最前面的多余 0——100 万位 π 的输出就是这么直接得来的。

4. 加法与进位

原理。 小学的「逐位相加、逢千进一」是天生串行的（第 k 位进位要等第 k−1 位）。GPU 上改成两步： 先整片相加（某些 limb 会暂时超过 999，不管它），再整片传播进位——把每个 limb 的溢出 c // 1000 整体挪高一格加上去、自己留 c % 1000，重复几趟直到没有溢出。每趟都是整片数组运算。

代码。 进位归一化，以及基于它的加法：

def normalize(c):
    """把（可能含很大项的）系数张量归一到每个 limb 都在 [0, B) 的规范形式。
    向量化的进位传播：每一趟把溢出部分整体上移一位相加，直到没有进位。"""
    c = F.pad(c, (0, 8))  # 预留高位空间容纳进位
    while True:
        carry = c // B
        if int(torch.count_nonzero(carry).item()) == 0:
            break
        c = c % B
        c = c + F.pad(carry[:-1], (1, 0))  # carry[i] 加到 limb i+1
    return strip(c)
def add(x, y):
    n = max(x.shape[0], y.shape[0])
    x = F.pad(x, (0, n - x.shape[0]))
    y = F.pad(y, (0, n - y.shape[0]))
    return normalize(x + y)

详解。 normalize 里 carry = c // 1000 一次算出所有 limb 的溢出； c[1:] = c[1:] + carry[:-1] 把溢出整体挪高一位相加（这一行就是「进位」的并行版）。 一次乘法产生的系数可能高达几千亿，但 1000 的幂每趟把溢出缩小 1000 倍，通常 4～6 趟就全部归位； count_nonzero(carry) 用来判断「还有没有进位」。add 只需把两数补齐等长、相加，再交给 normalize。

减法用对称的「借位」思路（下面附了 sub 的实现），比较大小 cmp 则从最高位找第一个不同的 limb。

def sub(x, y):
    """返回 x - y，要求 x >= y >= 0。向量化借位传播。"""
    n = max(x.shape[0], y.shape[0])
    x = F.pad(x, (0, n - x.shape[0]))
    y = F.pad(y, (0, n - y.shape[0]))
    d = x - y
    while True:
        neg = d < 0
        if int(torch.count_nonzero(neg).item()) == 0:
            break
        negi = neg.to(DT)
        d = d + negi * B                       # 负 limb 加 B 变正
        d = d - F.pad(negi[:-1], (1, 0))       # 向高一位借 1
    return strip(d)

5. Chudnovsky 公式：每一项顶 14 位

原理。 莱布尼茨级数极慢（100 万项才 6～7 位）：

π4=1−13+15−17+⋯

我们改用 1988 年的 Chudnovsky 公式：

π=426880⋅10005∑k=0∞(6k)!(13591409+545140134k)(3k)!(k!)3(−640320)3k

关键事实：每多算一项，多确定约 14.18 位十进制（相邻项相差约 640320³ ≈ 2.6×1017 倍）。 所以 100 万位只需约 70,500 项；我们取 N = 71,432 并留 32 位保护位。

代码。 每一项（「叶子」）就是几个整数相乘。为避免 7 万次逐个上传，我们先在 GPU 上批量算好所有项的整型中间量 （Pval, a3, factor 都是长度 N 的张量），再按需取用：

    def leaf(a):
        if a == 0:
            return from_int(1), from_int(1), (1, from_int(A_CONST))
        P = i64_to_limbs(Pval[a], 6)          # (6a-5)(2a-1)(6a-1) <= 2.6e16 -> <=6 limbs
        Q = mul(i64_to_limbs(a3[a], 5), CONST)
        T = mul_scalar_t(P, factor[a])
        Ts = -1 if (a & 1) else 1
        return P, Q, (Ts, T)

详解。 对第 a 项：P=(6a−5)(2a−1)(6a−1)；Q=a³·10939058860032000（即 640320³/24）； T=P·(13591409+545140134a)，奇数项取负号。其中 Q 高达约 4×10³⁰，超过 int64——所以走大整数乘法 mul(...) 得到多 limb 结果；a=0 是特例（P=Q=1, T=13591409）。这些张量索引 Pval[a] 都发生在 GPU 上。

6. 二分裂：把「求和」变成少数几次乘法

原理。 7 万个分数直接通分相加，分母会爆炸、要做 O(N²) 次大数加法。二分裂像归并排序一样分而治之： 一段连续项的和写成一个分数 T/Q（外加辅助量 P），相邻两段合并只需几次乘法。全程精确整数，无舍入。

递归对半拆、自底向上合并。

相邻两段（中点 m）的合并规则是：

P(a,b)=P(a,m)⋅P(m,b)Q(a,b)=Q(a,m)⋅Q(m,b)T(a,b)=Q(m,b)⋅T(a,m)+P(a,m)⋅T(m,b)

代码。 递归与合并规则；T 可能为负，用「幅值 + 符号」表示，合并时做有符号加法 _sadd：

    def bs(a, b):
        if b - a == 1:
            return leaf(a)
        m = (a + b) // 2
        Pam, Qam, Tam = bs(a, m)
        Pmb, Qmb, Tmb = bs(m, b)
        P = mul(Pam, Pmb)
        Q = mul(Qam, Qmb)
        s1, m1 = Tam
        s2, m2 = Tmb
        t1 = mul(Qmb, m1)          # 符号 s1
        t2 = mul(Pam, m2)          # 符号 s2
        Tmag, Ts = _sadd(s1, t1, s2, t2)
        return P, Q, (Ts, Tmag)
def _sadd(s1, m1, s2, m2):
    """有符号加：(s1*m1) + (s2*m2) -> (magnitude, sign)。"""
    if s1 == s2:
        return add(m1, m2), s1
    c = cmp(m1, m2)
    if c == 0:
        return zero(), 1
    if c > 0:
        return sub(m1, m2), s1
    return sub(m2, m1), s2

详解。 合并公式 P=Pₐₘ·Pₘᵦ, Q=Qₐₘ·Qₘᵦ, T=Qₘᵦ·Tₐₘ+Pₐₘ·Tₘᵦ 每次只需 4 次乘法， 而且总是「两个规模相近的数」相乘——这对下一节的 FFT 乘法最友好。递归到底就是上一节的叶子。 最终得到两个精确大整数 Q(0,N) 与 T(0,N)（各约 200 万位）。

7. FFT 大整数乘法（最需要 GPU 的一步）

原理。 把大数看成「1000 进制多项式」，两数相乘＝系数做卷积。朴素卷积 O(n²)，对 33 万 limb 要 1011 次乘法，太慢。 卷积定理说：在「频域」里卷积＝逐点相乘。于是走一条近路——FFT 变到频域、逐点乘、逆 FFT 变回来，总代价降到 O(n log n)。

FFT 乘法：把难做的卷积变成好做的逐点相乘。

代码。 小数字用直接的「外积求和」小乘法（避免 FFT 启动开销），大数字才用 torch.fft：

def mul_school(x, y):
    """小规模乘法：外积后按反对角线累加（在 GPU 上）。"""
    nx, ny = x.shape[0], y.shape[0]
    outer = (x.view(-1, 1) * y.view(1, -1)).reshape(-1)
    ii = torch.arange(nx, device=DEV).view(-1, 1).expand(nx, ny)
    jj = torch.arange(ny, device=DEV).view(1, -1).expand(nx, ny)
    idx = (ii + jj).reshape(-1)
    coeffs = torch.zeros(nx + ny - 1, dtype=DT, device=DEV)
    coeffs.index_add_(0, idx, outer)
    return normalize(coeffs)
def mul_fft(x, y):
    """大规模乘法：float64 FFT 做线性卷积，四舍五入回整数，再归一化。"""
    nx, ny = x.shape[0], y.shape[0]
    m = nx + ny - 1
    s = _npow2(m)
    xf = torch.fft.rfft(x.to(torch.float64), n=s)
    yf = torch.fft.rfft(y.to(torch.float64), n=s)
    z = torch.fft.irfft(xf * yf, n=s)[:m]
    zr = torch.round(z)
    err = float((z - zr).abs().max().item())
    assert err < FFT_ASSERT, "FFT rounding error too large: %g (limbs %d x %d)" % (err, nx, ny)
    return normalize(zr.to(DT))
def mul(x, y):
    nx, ny = x.shape[0], y.shape[0]
    if nx == 0 or ny == 0:
        return zero()
    if max(nx, ny) <= SCHOOL_THRESH:
        return mul_school(x, y)
    return mul_fft(x, y)

详解。 rfft/irfft 就是上图的 ① 和 ③，中间 xf*yf 是逐点相乘②；PyTorch 的这几个函数直接跑在 GPU 上。 FFT 内部用浮点，为何还能得到精确整数？因为 float64 能精确表示到 253≈9×1015， 而我们的系数最大约 n·999² ≈ 7×1011，还差约一万倍余量，四舍五入 round() 就能无误差还原。 更关键的是那行安全阀 assert err < 0.25：每次乘法都检查浮点结果离最近整数有多远，一旦越界立即报错， 绝不悄悄算错。整个 100 万位计算里它从未触发。

8. 牛顿迭代：只靠乘法做出除法与开方

原理。 组装 π 需要一次开方和一次大整数除法，但我们只想用已经造好的乘法。牛顿迭代做到了： 求倒数 1/T 用如下迭代，它二次收敛——每迭代一次正确位数翻倍：

xn+1=xn⁢(2−T⋅xn)

从一个 double 种子（约 15 位）起步，约 17 步就到 100 万位。开方同理（先求 1/√K 再乘回 K）：

yn+1=yn⁢(3−K⋅yn2)2

其中要「除以 2」。因为基 1000 是偶数，除以 2 有个完全并行的写法——每个 limb 一次算完，无需串行借位：

def half(x):
    d = x
    lower = d // 2
    parity_next = F.pad((d % 2)[1:], (0, 1))   # d_{i+1} 的奇偶性挪到位置 i
    res = lower + (B // 2) * parity_next
    return strip(res)

详解（取半）。 第 i 个 limb 的结果为：

ri=⌊di/2⌋+500⋅(di+1mod2)

因为 1000 是偶数，向下传的借位恰好就是「高一位 limb 的奇偶性」di+1 mod 2， 所以所有 limb 能一次并行算出，结果都 < 1000，无需再传播。

代码（除法）。 牛顿倒数 _recip 求 ⌊1000g/T⌋，divide 用它算 ⌊A/T⌋ 并做精确 ±1 校正：

def _recip(T, g):
    Ts = strip(T)
    dt = Ts.shape[0]
    assert g >= dt
    # 用最高 2 个 limb 估计种子：T ≈ top * B**(dt-2)
    if dt >= 2:
        top = int(Ts[-1].item()) * B + int(Ts[-2].item())
        e_top = dt - 2
    else:
        top = int(Ts[-1].item())
        e_top = dt - 1
    # w0 ≈ B**g / T = B**(g-e_top) / top ；用 q=B**4//top 放到合适位置
    q = (B ** 4) // top
    shift = g - e_top - 4
    if shift >= 0:
        w = shl_limbs(from_int(q), shift)
    else:
        w = from_int(q // (B ** (-shift)))
    Bg = one_at(g)
    one = from_int(1)
    target_digits = max(1, (g - dt + 1) * LOGB)
    niters = int(math.ceil(math.log2(target_digits / 4.0 + 2))) + 4
    for _ in range(niters):
        Tw = mul(T, w)
        c = cmp(Bg, Tw)
        if c == 0:
            break
        if c > 0:
            e = sub(Bg, Tw); es = 1
        else:
            e = sub(Tw, Bg); es = -1
        delta = shr_limbs(mul(w, e), g)
        if int(torch.count_nonzero(delta).item()) == 0:
            break
        w = add(w, delta) if es > 0 else sub(w, delta)
    # 精确 ±修正：保证 T*w <= B**g < T*(w+1)
    guard = 0
    while cmp(mul(T, w), Bg) > 0:
        w = sub(w, one); guard += 1
        assert guard < 8
    guard = 0
    while cmp(mul(T, add(w, one)), Bg) <= 0:
        w = add(w, one); guard += 1
        assert guard < 8
    return w
def divide(A, T):
    """返回 floor(A / T)，A、T 为非负 limb 张量，T>0。"""
    A = strip(A); T = strip(T)
    if cmp(A, T) < 0:
        return zero()
    g = A.shape[0] + 2
    w = _recip(T, g)
    q = shr_limbs(mul(A, w), g)
    one = from_int(1)
    # ±修正：保证 q*T <= A < (q+1)*T
    guard = 0
    while cmp(mul(q, T), A) > 0:
        q = sub(q, one); guard += 1
        assert guard < 8
    guard = 0
    while cmp(mul(add(q, one), T), A) <= 0:
        q = add(q, one); guard += 1
        assert guard < 8
    return strip(q)

详解（除法）。 _recip 先用 T 的最高两位 limb 估一个种子（约 5～6 位准），然后迭代 w ← w + w·(1000g − T·w)/1000g（其中「除以 1000g」就是把低 g 个 limb 整段丢掉，是精确的移位）； 迭代次数按目标位数取（约 log₂），末尾用几步 ±1 把「取整方向」校正到绝对正确。 divide 拿到倒数后乘一下、再移位，同样做 ±1 校正，保证 q·T ≤ A < (q+1)·T。

代码（开方）。 倒数平方根牛顿 y ← y·(3 − K·y²)/2，最后 s = ⌊K·y⌋ ≈ √K，再 ±1 校正：

def isqrt_big(K):
    K = strip(K)
    dk = K.shape[0]
    if dk == 1:
        return from_int(math.isqrt(int(K[0].item())))
    h = dk + 2
    # 种子：取最高 kt 个 limb（保证 e_low = dk-kt 为偶，便于开方对齐）
    kt = 6 if (dk - 6) % 2 == 0 else 5
    kt = min(kt, dk)
    if (dk - kt) % 2 != 0:
        kt -= 1
    e_low = dk - kt
    Kh = 0
    for i in range(kt):
        Kh = Kh * B + int(K[dk - 1 - i].item())   # 最高 kt 个 limb 的值
    # K ≈ Kh * B**e_low, sqrt(K) ≈ sqrt(Kh) * B**(e_low/2)
    r = 1.0 / math.sqrt(float(Kh))                 # ≈ 1/sqrt(Kh) ∈ (0,1)
    mant = int(round(r * (B ** 6)))                # r ≈ mant * B**-6
    # y0 = B**h / sqrt(K) ≈ mant * B**-6 * B**(h - e_low/2)
    shift = h - (e_low // 2) - 6
    if shift >= 0:
        y = shl_limbs(from_int(mant), shift)
    else:
        y = from_int(mant // (B ** (-shift)))
    B2h = one_at(2 * h)
    target_digits = max(1, h * LOGB)
    niters = int(math.ceil(math.log2(target_digits / 4.0 + 2))) + 5
    for _ in range(niters):
        Ky2 = mul(K, mul(y, y))
        c = cmp(B2h, Ky2)
        if c == 0:
            break
        if c > 0:
            rr = sub(B2h, Ky2); rs = 1
        else:
            rr = sub(Ky2, B2h); rs = -1
        delta = half(shr_limbs(mul(y, rr), 2 * h))
        if int(torch.count_nonzero(delta).item()) == 0:
            break
        y = add(y, delta) if rs > 0 else sub(y, delta)
    s = shr_limbs(mul(K, y), h)
    one = from_int(1)
    guard = 0
    while cmp(mul(add(s, one), add(s, one)), K) <= 0:
        s = add(s, one); guard += 1
        assert guard < 16
    guard = 0
    while cmp(mul(s, s), K) > 0:
        s = sub(s, one); guard += 1
        assert guard < 16
    return strip(s)

详解（开方）。 内层循环全是乘法（跑在 GPU 上的 FFT 乘法）加上那个并行的「除以 2」half； 末尾 while (s+1)² ≤ K / while s² > K 各校正至多几步，确保 s 恰为 ⌊√K⌋。

9. 组装出 π

原理。 二分裂给出精确整数 Q、T 后，把 π 放大 10p 倍取整：

π⋅10p=⌊426880⋅Q(0,N)⋅10005⋅102pT(0,N)⌋

因为基是 1000＝10³，102p 恰好是整数个 limb（这也是取 p 为 3 的倍数的原因），开方与移位都精确。

代码。 组装的关键几行：

    # --- sqrt(10005) 放大到 10^p：isqrt(10005 * 10^{2p}) = isqrt(10005 * B^{2p/3}) ---
    K = shl_limbs(from_int(C_RAD), (2 * p) // LOGB)
    sqrtC = isqrt_big(K)
    if device == "cuda":
        torch.cuda.synchronize()
    t_sqrt = time.time()
    if verbose:
        print("  [isqrt(10005*1e2p)]      %.2fs" % (t_sqrt - t_bs))

    # --- π*10^p = floor(426880 * Q * sqrtC / T) ---
    num = mul_scalar(Q, C_LIN)
    num = mul(num, sqrtC)
    pi_scaled = divide(num, T)

详解。 shl_limbs(from_int(10005), 2p/3) 造出 10005·102p（在低位补零＝乘 1000 的幂）； isqrt_big 得到 √10005·10p；再乘上 426880·Q、除以 T，就是 π·10p。 最后 to_decimal_str 把它转成十进制字符串，取「3.」加上后 100 万位即为结果。

10. 与公开数据逐位比对

原理。 从公开来源下载 π 的百万位权威数据，先校验这份数据本身可信（前 50 位、以及第 762–767 位著名的「费曼点」六个 9）， 再把我们 GPU 算出的全部 100 万位逐字符比对。
